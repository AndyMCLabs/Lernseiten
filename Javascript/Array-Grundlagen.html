Array Grundlagen

Willkommen in der Welt der Arrays! Stell dir ein Array vor wie einen magischen Beutel in deinem Inventar: Du kannst dort alle möglichen Dinge aufbewahren - Tränke, Items, Zahlen, Texte oder sogar andere Beutel! 🎒✨

Was ist ein Array?

Ein Array ist eine geordnete Sammlung von Werten. Es ist wie eine Liste, bei der jedes Element eine Position (Index) hat. In JavaScript beginnt der Index immer bei 0. Das erste Element ist also an Position 0, das zweite an 1, usw.

// Unser Helden-Inventar als Array
const inventar = ["Schwert", "Heiltrank", "Gold", "Zauberbuch"];
console.log(inventar[0]); // "Schwert"
console.log(inventar[2]); // "Gold"

Arrays erstellen

Es gibt zwei Möglichkeiten, ein Array zu erstellen:

// Mit eckigen Klammern (empfohlen)
const zahlen = [1, 2, 3];

// Mit dem Array-Konstruktor (selten verwendet)
const andereZahlen = new Array(1, 2, 3);

Elemente lesen und ändern

const held = ["Name", 100, "Krieger"];
console.log(held[1]); // 100 (Leben)

// Wir rüsten den Helden auf!
held[1] = 150; // Mehr Lebenspunkte
console.log(held); // ["Name", 150, "Krieger"]

Länge eines Arrays

const skills = ["Feuerball", "Blitz", "Heilen"];
console.log(skills.length); // 3

Best Practices

Best Practices

✅ Nutze sprechende Namen für deine Arrays
✅ Verwende const, wenn sich das Array selbst nicht ändern soll
✅ Indexe immer mit Bedacht nutzen - denk daran, sie starten bei 0!
✅ Nicht über das Ende des Arrays hinausgreifen - das ergibt undefined
✅ Benutze .length, um Array-Längen dynamisch zu nutzen
✅ Nutze Array-Methoden wie map(), filter() und reduce(), um Code eleganter und funktionaler zu gestalten
✅ Vermeide "magische Zahlen" im Code - verwende aussagekräftige Variablen für Array-Längen oder Grenzen
✅ Wenn du nur prüfen willst, ob ein Element vorhanden ist, nimm includes() - das liest sich wie ein Zauberspruch! 🪄
✅ Nutze for...of oder forEach() statt klassischer Schleifen, wenn du nicht mit Indexen arbeitest - das macht deinen Code lesbarer
✅ Kommentiere komplexe Array-Manipulationen - besonders wenn du mit verschachtelten Arrays (Arrays in Arrays) arbeitest, damit dein künftiges Ich nicht flucht! 😅

Fragen mit Antworten

  1. Was ist der Index des ersten Elements in einem Array?
    A: 0
  2. Wie viele Elemente hat das Array [1,2,3,4]?
    A: 4
  3. Wie kann man das zweite Element eines Arrays ändern?
    A: Mit array[1] = neuerWert
  4. Was gibt array[100] zurück, wenn das Array nur 3 Elemente hat?
    A: undefined
  5. Wie kann man ein leeres Array erstellen?
    A: const leer = [];
  6. Wie fügt man ein Element am Ende eines Arrays hinzu?
    A: Mit array.push(element)
  7. Wie entfernt man das letzte Element eines Arrays?
    A: Mit array.pop()
  8. Wie kann man überprüfen, ob ein Wert im Array enthalten ist?
    A: Mit array.includes(wert)
  9. Wie erhält man die Länge eines Arrays?
    A: Mit array.length
  10. Wie kann man ein Array umdrehen?
    A: Mit array.reverse()

Häufige Fehler und Lösungen

  • Fehler: Zugriff auf ein Element mit einem negativen Index → Lösung: Nur positive Zahlen oder 0 nutzen
  • Fehler: Element außerhalb der Länge aufrufen → Lösung: Immer prüfen mit .length
  • Fehler: Array mit falschem Konstruktor erstellen → Lösung: Nutze [] statt new Array()
  • Fehler: Änderung mit const nicht möglich → Lösung: Du kannst die Inhalte ändern, nur nicht die Referenz
  • Fehler: Mix aus Datentypen macht Verarbeitung schwierig → Lösung: Wenn möglich, gleichartige Datentypen nutzen
  • Fehler: Methode auf ein leeres Array anwenden → Lösung: Vorher mit array.length > 0 prüfen
  • Fehler: for...in für Arrays verwenden → Lösung: Nutze for, forEach() oder for...of für Arrays
  • Fehler: Array versehentlich kopiert mit Referenz → Lösung: Für echte Kopien [...array] oder array.slice() verwenden
  • Fehler: Methoden wie push() mit falscher Syntax → Lösung: Immer runde Klammern nutzen, z.B. array.push(42)
  • Fehler: undefined bei nicht existierenden Indizes → Lösung: Vor dem Zugriff prüfen: if (array[index] !== undefined)

Speed Debugging Challenges

  1. Ein Array gibt undefined zurück - was wurde falsch gemacht?
  2. Ein Spiel soll ersetzt werden, aber array[2] funktioniert nicht - warum?
  3. Die Länge ist falsch - wurde das Array richtig erstellt?
  4. Warum lässt sich der Wert im Array nicht ändern? (Ist das Array eventuell konstant?)
  5. Ein Array-Element ist plötzlich null - wo könnte der Fehler im Code liegen?
  6. Beim Zugriff auf das Array kommt ein Fehler: TypeError: array is not iterable - was ist passiert?
  7. Die Schleife läuft zu oft oder zu selten - wurde die Array-Länge korrekt verwendet?
  8. Ein Array wird als leer angezeigt, obwohl Elemente drin sein sollten - wann passiert das?
  9. Ein Array enthält unerwartete Daten - wo könnte eine falsche Zuweisung stattgefunden haben?
  10. Warum wird ein Objekt im Array nicht richtig dargestellt? Liegt es an der Referenz oder Kopie?
