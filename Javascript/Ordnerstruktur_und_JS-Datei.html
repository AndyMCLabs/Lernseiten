Ordnerstruktur & die JavaScript-Datei

Bevor du mit dem Coden beginnst, solltest du wie ein echter Code-Ranger deine Basis aufbauen: Ein klar strukturierter Projektordner hilft dir, Ordnung zu bewahren und Bugs den Gar aus zu machen!

Wie sieht eine gute Ordnerstruktur aus?

Hier ein klassisches Grundgerüst für kleine bis mittlere Projekte:

projekt-name/
├── index.html         
├── css/
│   └── style.css      
├── js/
│   └── script.js      
└── assets/
└── bilder/

Die JavaScript-Datei liegt in einem eigenen js-Ordner. Warum? Weil Struktur sexy ist. Und weil du später vielleicht noch weitere Skripte brauchst - für Magie, Kämpfe oder Interface-Kram.

Wie bindest du deine JavaScript-Datei ein?

In deiner index.html sieht das so aus:

<!DOCTYPE html>
<html lang="de">
    <head>
        <meta charset="UTF-8">
        <title>Mein JS-Projekt</title>
        <link rel="stylesheet" href="css/style.css">
    </head>
    <body>

        <h1>Hello JavaScript!</h1>

        <script src="js/script.js"></script>
    </body>
</html>

✨ Praktisches Beispiel

Deine script.js könnte so starten:

// js/script.js

console.log("🧙‍♀️ Deine JavaScript-Reise beginnt genau... JETZT!");
alert("Willkommen in der Welt von JavaScript!");

Best Practices für Struktur & Skripte

🌟 Benenne deine Dateien sprechend (nicht kram1.js oder test_final_2_END.js)
🌟 Halte deine Verzeichnisstruktur flach und logisch
🌟 Trenne HTML, CSS und JavaScript klar
🌟 Nutze relative Pfade, z.B. js/script.js
🌟 Binde JS am Ende des <body> ein, damit erst der HTML-Inhalt geladen wird

🧠 Fragen mit Antworten

  1. Wo speichert man normalerweise JavaScript-Dateien?
    In einem eigenen Ordner namens js.
  2. Wie wird eine JS-Datei in HTML eingebunden?
    Mit <script src="js/script.js"></script>.
  3. Warum sollte man HTML, CSS und JS trennen?
    Weil Ordnung und Übersichtlichkeit Bugs und Frust vermeiden.
  4. Was passiert, wenn du eine Datei falsch verlinkst?
    Der Browser findet sie nicht und du bekommst Errors in der Konsole.
  5. Was ist der Unterschied zwischen relativen und absoluten Pfaden?
    Relative Pfade zeigen relativ zum aktuellen Dokument, absolute Pfade ab Root oder Domain.

💪 Übungen zum Strukturieren & Einbinden

  1. Erstelle einen Projektordner mit Unterordnern: css, js, assets.
  2. Lege in js/ eine Datei script.js an und schreibe ein console.log().
  3. Verlinke die Datei korrekt in index.html und überprüfe sie im Browser.
  4. Füge eine zweite JavaScript-Datei ein (z.B. helper.js) und teste die Einbindung.
  5. Nutze alert(), um beim Laden der Seite eine Nachricht anzuzeigen.
