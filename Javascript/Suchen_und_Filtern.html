Suchen und Filtern - Finde die Nadel im Heuhaufen (aka Array)

Willkommen, Abenteurerin der Arrays! Heute lernst du, wie du mit der Präzision eines Jedi und dem Spürsinn eines Hexers gezielt Elemente aus einem Array aufspürst oder gefilterte Listen zauberst. Wir sprechen über die Methoden find(), filter(), includes() und indexOf() - mächtige Zauber für jede Inventarsuche!

Erklärung der Methoden

  • find(): Gibt das **erste** Element zurück, das einem Kriterium entspricht.
  • filter(): Gibt **alle** passenden Elemente als neues Array zurück.
  • includes(): Prüft, ob ein bestimmter Wert **im Array enthalten** ist - true oder false.
  • indexOf(): Gibt den **Index** des ersten Vorkommens zurück - oder -1, wenn nichts gefunden wurde.
const helden = ["Arthas", "Jaina", "Thrall", "Sylvanas"]; // find() const heldin = helden.find(name => name.startsWith("J")); console.log(heldin); // "Jaina" // filter() const rebellen = helden.filter(name => name.includes("a")); console.log(rebellen); // ["Arthas", "Jaina", "Sylvanas"] // includes() console.log(helden.includes("Illidan")); // false // indexOf() console.log(helden.indexOf("Thrall")); // 2

Best Practices

✅ Verwende find(), wenn du **nur ein** Element suchst.
✅ Verwende filter(), wenn du eine neue Gruppe willst.
✅ includes() ist perfekt für schnelle Prüfungen.
✅ indexOf() hilft dir, Positionen zu finden - für spätere Manipulationen.
✅ Nutze Pfeilfunktionen für elegante, kurze Bedingungen.

Praktisches Beispiel zum Nachmachen

Ein Inventarsystem für einen magischen Rucksack:

const inventar = [ { name: "Heiltrank", typ: "Trank" }, { name: "Feuerball-Zauberrolle", typ: "Zauber" }, { name: "Schild der Stärke", typ: "Ausrüstung" } ];
// Finde das erste Zauber-Item
const zauber = inventar.find(item => item.typ === "Zauber");
console.log("Zauber gefunden:", zauber);

// Filtere alle Tränke
const tränke = inventar.filter(item => item.typ === "Trank");
console.log("Tränke im Inventar:", tränke);

5 Fragen mit Antworten

  1. Was macht find()?
    Es gibt das erste passende Element zurück.
  2. Worin liegt der Unterschied zu filter()?
    filter() gibt ein Array mit allen passenden Werten zurück, find() nur das erste.
  3. Was passiert, wenn includes() nichts findet?
    Es gibt false zurück.
  4. Wann bekommst du bei indexOf() eine -1?
    Wenn das gesuchte Element nicht im Array enthalten ist.
  5. Kann man auch nach Objekten mit includes() suchen?
    Nur wenn du exakt das gleiche Objekt referenzierst - nicht mit Vergleich auf Inhalt.

5 häufige Fehler und Lösungen

  1. Fehler: find() gibt undefined zurück.
    Lösung: Prüfe dein Filterkriterium - vielleicht stimmt der Ausdruck nicht.
  2. Fehler: includes() findet ein Objekt nicht.
    Lösung: Objekte können nur mit derselben Referenz gefunden werden.
  3. Fehler: filter() verändert das ursprüngliche Array.
    Lösung: Nein, tut es nicht! Aber wenn du das neue Array nicht speicherst, ist es weg.
  4. Fehler: Vergessen, den Rückgabewert zu verwenden.
    Lösung: Wechsle zu return bei der Bedingung in find() oder filter().
  5. Fehler: Zugriff auf eine Eigenschaft von undefined nach find().
    Lösung: Prüfe, ob das Ergebnis existiert (if (result)).

Zusammenfassung

Ob du nach einem legendären Item oder dem verlorenen Charakter „Bob“ suchst - mit find(), filter(), includes() und indexOf() hast du magische Suchwerkzeuge in der Tasche. Wähle mit Bedacht, wie du suchst: Einzeltreffer, Listen, Prüfung oder Position - es gibt für alles das passende Tool.
