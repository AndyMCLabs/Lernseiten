Inline vs. Externes JavaScript - Das Duell der Codeformen

Stell dir vor, dein JavaScript wäre ein Superheld. Inline-JS ist wie der Typ, der mitten im Kampf noch sein Cape anzieht - er ist da, aber es ist... unordentlich. Externes JS dagegen kommt ausgerüstet, organisiert und bereit für alles - wie Batman mit Utility-Belt. 🦸‍♂️

Was ist Inline-JavaScript?

Inline-JS bedeutet: Du schreibst JavaScript **direkt in dein HTML**, meist innerhalb von Event-Attributen oder einem <script>-Tag direkt in der Seite.

<button onclick="alert('Hallo Held!')">Klick mich!</button>

Das ist schnell, aber schnell wird's auch unübersichtlich. Wie ein Inventar voller Items ohne Sortierung. 😵‍💫

Was ist Externes JavaScript?

Hier wird dein Code **in einer eigenen Datei** gespeichert - sauber, wartbar, wiederverwendbar. Du bindest die Datei dann mit <script src="main.js"></script> ein. Bäm, Ordnung! 

// main.js
alert('Organisierter Held meldet sich!');

Best Practises

✅ Verwende **externes JS**, um Code sauber zu halten und zu trennen.
✅ Nutze **Inline-JS** nur für schnelle Tests oder Mini-Demos.
✅ Lagere Event-Handler aus und verwende addEventListener statt onclick direkt im HTML.
✅ Trenne Struktur (HTML), Präsentation (CSS) und Verhalten (JS) - das ist der Dreifaltigkeits-Code-Glaube! ✝️💻✨

Beispiel zum Nachmachen

HTML-Datei:

<button id="heroBtn">Sag Hallo!</button>
<script src="main.js" defer></script>

main.js:

document.getElementById("heroBtn").addEventListener("click", () => {
alert("Hallo, mutiger Coder!");
});

5 Quizfragen (mit Antworten)

  1. Was ist Inline-JavaScript?
    A: JavaScript direkt im HTML.
  2. Was ist externer JavaScript-Code?
    A: JavaScript in einer ausgelagerten Datei.
  3. Warum ist externes JS besser wartbar?
    A: Weil es übersichtlicher und wiederverwendbar ist.
  4. Was bedeutet Separation of Concerns?
    A: HTML, CSS und JS sollen getrennt sein.
  5. Was ist sicherer: Inline oder externes JS?
    A: Externes JS - leichter zu kontrollieren und auditieren.

5 Übungen

  1. Erstelle einen Button mit Inline-JS, der eine Nachricht zeigt.
  2. Schreibe die gleiche Funktion in einer externen main.js-Datei.
  3. Füge defer hinzu und beobachte, wann der Code ausgeführt wird.
  4. Ersetze Inline-Eventhandler durch addEventListener.
  5. Organisiere deinen Code in Module - z.B. für Buttons, Inputs und Animationen (Fortgeschritten).

5 Projektideen

  1. Ein „Klick mich!“-Button, der bei jedem Klick etwas anderes sagt.
  2. Eine To-Do-Liste mit externem Script und DOM-Manipulation.
  3. Ein Light/Dark-Mode-Toggle mit externer Datei.
  4. Ein Emoji-Würfler mit zufälliger Ausgabe per externem JS.
  5. Ein Minispiel: „Fang den Button“ - bewegt sich weg, wenn du drüberfährst.
