Namen und Konventionen - Der Stil deiner Variablen!

Ah, du bist also bereit, die geheime Kunst des „Benennt-ismus“ zu meistern! 🧙‍♀️ Denn so wie ein Held einen ehrenvollen Namen braucht, brauchen auch Variablen, Funktionen und Co. in JavaScript Namen, die ihren Zweck klar machen - für dich, deine Mitstreiter und dein zukünftiges Ich (das beim Code-Lesen flucht oder feiert 😅).

Erklärung

Gute Namen in der Programmierung sind Gold wert! Sie machen den Code lesbar, wartbar und geben sofort eine Vorstellung davon, was passiert. In JavaScript (und fast jeder modernen Sprache) gelten bestimmte **Konventionen**, also „Stilregeln“ für die Namensvergabe:

1. camelCase für Variablen und Funktionen

  • spielStarten, anzahlLeben, berechnePunkte
  • Erster Buchstabe klein, jedes neue Wort mit großem Anfangsbuchstaben.

2. PascalCase für Konstruktoren & Klassen

  • Spieler, ZauberTrank
  • Jedes Wort beginnt mit einem Großbuchstaben - wie bei Heldennamen!

3. GROSSBUCHSTABEN_UND_UNDERSCORES für Konstanten (optional)

  • MAX_LEBEN, API_URL
  • Nur bei Werten, die sich nie ändern - wie Naturgesetze 🧪

4. Keine Sonderzeichen, keine Umlaute

  • größe, anzahl-der-leben = ❌
  • Nur Buchstaben, Zahlen, $ und _ sind erlaubt.

5. Namen sollen beschreibend, nicht kryptisch sein

  • let x = 42; = 🤔
  • let punkteProLevel = 42; = 👍

Best Practices

  • Vermeide zu kurze Namen (x, abc), es sei denn für temporäre Zähler.
  • Nutze sprechende Namen - der Code erklärt sich dann fast von selbst.
  • Vermeide Sonderzeichen und Leerzeichen - das gibt sonst Fehler.
  • Keine Umlaute - auch wenn's deutsch ist. 😉
  • Verwende eine einheitliche Schreibweise im ganzen Projekt!

Beispiel

// ❌ Schlechte Beispiele:
let x = 5;
const y = "Spieler";
function z() {
return x * 2;
}

// ✅ Gute Beispiele:
let anzahlLeben = 5;
const SPIELER_NAME = "Zelda";
function berechneMultiplikator(wert) {
return wert * 2;
}

5 Fragen mit Antworten

  1. Was ist camelCase?
    Ein Stil, bei dem der erste Buchstabe klein ist und jedes neue Wort mit Großbuchstaben beginnt, z. B. spielStarten.
  2. Warum sollte man keine Umlaute in Variablennamen verwenden?
    Weil sie in vielen Programmierumgebungen Probleme machen können.
  3. Wann benutzt man PascalCase?
    Für Klassen oder Konstruktorfunktionen wie NeuerSpieler.
  4. Ist let Leben = 3; ein guter Name?
    Fast! Der Anfangsbuchstabe sollte klein sein: leben.
  5. Was ist an let x = 42; problematisch?
    Der Name sagt nichts über den Zweck der Variable aus.

5 Häufige Fehler und Lösungen

  • Fehler: Umlaute in Variablennamen
    Lösung: Verwende „ae“, „oe“, „ue“ statt „ä“, „ö“, „ü“.
  • Fehler: Namen mit Leerzeichen
    Lösung: Verwende camelCase oder _.
  • Fehler: Gemischte Namensstile im Projekt
    Lösung: Halte dich an einheitliche Konventionen!
  • Fehler: Verwendung von reservierten Wörtern wie let let = 3;
    Lösung: Nutze keine Schlüsselwörter als Namen.
  • Fehler: Knappe, kryptische Namen wie tmp
    Lösung: Schreibe lieber zwischenSpeicher oder tempDatei.

5 Übungen

  1. Benenne drei Variablen sinnvoll für ein Rollenspiel: Spielername, Lebenspunkte, Magielevel.
  2. Schreibe eine Funktion mit dem Namen berechneSchaden und gib darin einen festen Wert zurück.
  3. Erstelle eine Klasse Held mit einem Konstruktor, der name und level als Parameter nimmt.
  4. Erstelle eine const-Variable für den maximalen Mana-Wert mit richtigem Stil.
  5. Versuche, eine Variable mit einem reservierten Wort zu deklarieren - was passiert?

5 Projektideen

  1. Charakterblatt-Generator - Spieler gibt Namen und Werte ein, alles wird ordentlich benannt und angezeigt.
  2. Item-Benennungsmaschine - Namen und Eigenschaften von Items nach festen Namenskonventionen erstellen.
  3. Konventionen-Checker - Ein Tool, das deinen Code auf gute Benennungen prüft.
  4. Namens-Konverter - Gib einen Namen ein, der wird automatisch in camelCase oder PascalCase umgewandelt.
  5. Variablen-Ratespiel - Nutzer rät anhand des Namens, was die Variable wohl tut.

3 Speed Debugging-Challenges

  1. Challenge: Warum funktioniert let zauber-trank = 3; nicht?
    Tipp: Minuszeichen sind nicht erlaubt in Namen!
  2. Challenge: Was ist an let Rückgabe = 42; gefährlich?
    Tipp: Umlaute vermeiden - lieber rueckgabe!
  3. Challenge: Warum gibt console.log(SPIELERname) einen Fehler, wenn const spielerName = "Link"; ist?
    Tipp: JavaScript ist case-sensitive!

Zusammenfassung

Gute Namen sind wie legendäre Schwerter - sie helfen dir, besser zu kämpfen (äh, zu coden) ⚔️. Halte dich an Konventionen wie camelCase und benutze beschreibende Namen, die auch nach Wochen noch Sinn ergeben. So machst du dir (und deinem Team) das Leben leichter!
